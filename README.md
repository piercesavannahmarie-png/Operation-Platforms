## Portfolio Reflection – Software Design Document (The Gaming Room)

**Project Overview**

This project was completed for The Gaming Room, a client seeking to expand their existing Android game, *Draw It or Lose It*, into a scalable web-based platform. The goal was to design a distributed application capable of supporting multiple users, teams, and concurrent game sessions across platforms. The final deliverable was a comprehensive software design document outlining system requirements, architectural considerations, and platform recommendations for future expansion.

**Key Strengths**

A major strength of this project was my ability to translate technical concepts into clear, structured documentation suitable for both technical and non-technical stakeholders. I effectively applied object-oriented design principles and incorporated patterns such as Singleton-based service management to ensure consistency and scalability. I also provided a well-supported evaluation of operating environments and delivered a clear recommendation aligned with the client’s long-term goals.

**Value of the Design Process**

Developing this design document reinforced the importance of architectural planning prior to implementation. By defining requirements, constraints, and platform considerations early, I gained a deeper understanding of how distributed systems function and how design patterns influence scalability, maintainability, and reliability. This structured planning approach reduces development risk and supports more efficient implementation.

**Opportunities for Improvement**

If revisiting this project, I would enhance the platform evaluation by incorporating more quantitative comparisons, such as performance benchmarks or deployment case studies. Adding visual architecture diagrams would also improve clarity and make the document more accessible to stakeholders evaluating technical tradeoffs.

**User-Centered Design Perspective**

Throughout the project, I focused on aligning design decisions with the client’s core objectives: cross-platform accessibility, scalability, and consistent performance. These needs informed key recommendations such as adopting a Linux-based server environment and a distributed architecture. This experience reinforced the importance of user-centered design, as successful software must balance technical excellence with real-world usability and scalability requirements.

**Design Approach and Future Application**

My design approach emphasized structured analysis, modular thinking, and iterative refinement. I decomposed the system into logical components, evaluated platform tradeoffs, and applied object-oriented modeling to support long-term maintainability. Moving forward, I plan to continue using formal requirement analysis and architectural modeling while expanding my use of visual documentation and prototyping. These strategies will strengthen both technical decision-making and stakeholder communication in future software design work.
